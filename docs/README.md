## 🎯 文档使用指南

### 对于 AI 助手

当需要了解项目信息时，按以下优先级阅读文档：

#### 1. 快速了解项目 (5分钟)
- **阅读**: `QUICK_START.md`
- **获取**: 项目概述、快速启动、核心功能、常见问题

#### 2. 技术细节查询 (10-15分钟)
- **阅读**: `PROJECT_DOCUMENTATION.md` 对应章节
- **获取**: 详细的技术实现、API文档、配置说明、错误处理

#### 3. 架构设计理解 (20-30分钟)
- **阅读**: `ARCHITECTURE.md`
- **获取**: 系统架构、设计模式、安全机制、性能优化

## 💡 开发建议

### 对于新功能开发
1. 先了解现有架构和设计模式
2. 遵循模块化设计原则
3. 保持代码风格一致性
4. 添加适当的注释和文档

### 对于 Bug 修复
1. 先查看错误处理机制
2. 分析日志和错误信息
3. 参考常见问题解决方案
4. 测试修复效果

### 对于性能优化
1. 先了解性能优化架构
2. 分析瓶颈所在
3. 采用合适的优化策略
4. 监控优化效果

### 对于代码重构
1. 先理解架构设计思想
2. 遵循设计原则
3. 保持向后兼容
4. 充分测试
